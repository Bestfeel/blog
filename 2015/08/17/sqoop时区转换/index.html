sqoop 时区转换

在实际应用中，我们经常碰见各种时区转换问题。
在做 ETL 过程中，由于我们的数据时间是 0 时区的。需要转换 8 时区。

oracle 的处理方式

mysql 没有像 oracle 这么简单。需要自己转换。查找相关资料 mysql 有 2 种方式解决。

1. 将 mysql time_zone 设置成 +8，那么查询的时间字段将会自动加 8 小时。

2. 使用 CONVERT_TZ 函数转换
